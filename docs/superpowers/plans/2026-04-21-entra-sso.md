# Microsoft Entra SSO — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Microsoft Entra SSO as a second sign-in method, mandatory for trusted domains (`@covermesh.com` first; Ecclesia and SCHUNCK added via a one-file PR later). External client users keep email magic-link.

**Architecture:** One `MicrosoftEntraId` provider with the `/organizations/v2.0` issuer; a typed in-code tenant allowlist (`TID → { emailDomain, displayName }`) enforced in a `signIn` callback; `allowDangerousEmailAccountLinking: true` links existing magic-link users to their Entra identity on first SSO. Magic-link is gated off for trusted domains both at the UI and as a backstop in `sendMagicLink`.

**Tech Stack:** Auth.js v5 (next-auth beta), Drizzle ORM (Turso/libSQL), Next.js 16, React 19, Vitest, Resend.

**Spec:** `docs/superpowers/specs/2026-04-21-entra-sso-design.md`

## Implementation deviations from this plan

> Added after implementation to document intentional deviations. Read
> before relying on the task text below literally.

1. **Task 2 — gate location changed.** The plan placed the magic-link
   trusted-domain backstop inside `sendMagicLink` (sendVerificationRequest),
   with a test at `src/lib/auth/send-magic-link.test.ts`. A code review
   during implementation traced `@auth/core` and found that
   `sendVerificationRequest` thrown errors are NOT mapped to `AccessDenied`
   — only errors from `callbacks.signIn` are. The gate was moved to
   `authConfig.callbacks.signIn` in `src/lib/auth/config.ts`, tested in
   `src/lib/auth/config.test.ts`. `send-magic-link.ts` was reverted to
   its pre-Task-2 state.
2. **`pages.error: "/login"` added.** Without it, `AccessDenied` /
   `Verification` / `Configuration` errors redirected to Auth.js's
   built-in `/api/auth/error` page, bypassing the localized German copy
   in `src/app/login/page.tsx`. Fix committed as part of Task 2.
3. **Task 3 — send-magic-link import step skipped.** Because of (1),
   `send-magic-link.ts` no longer imports `isTrustedDomain`. The plan's
   step to update that import is obsolete.
4. **Task 4 — callback extended, not replaced.** Because (1) already
   introduced a `signIn` callback with an email branch, Task 4 added
   the Entra branch alongside it rather than creating the callback from
   scratch. Module augmentation was NOT needed — `src/types/next-auth.d.ts`
   already declared `role` and `clientId` on `User`.
5. **`vitest.config.ts` infra additions.** Testing `authConfig` under
   vitest required a `next/server` resolve alias plus inlining
   `next-auth` + `@auth/core` (Next 16's `next` package has no `exports`
   field, so vitest's ESM resolver can't follow the bare specifier).
   See commit `355cb25` for the rationale.

Everything else matches the plan as written.

```ts
// src/lib/auth/entra-tenants.ts
//
// Tenant allowlist for Microsoft Entra SSO.
// Adding a new tenant (Ecclesia, SCHUNCK, …):
//   1. Add a new env var (e.g. ENTRA_ECCLESIA_TENANT_ID)
//   2. Add a new entry in TENANT_ALLOWLIST() below
//   3. Have the tenant admin consent via the admin-consent URL (see docs/auth-entra-setup.md)

export type TenantConfig = {
  emailDomain: string;
  displayName: string;
};

export type EntraValidation =
  | { ok: true }
  | { ok: false; reason: "unknown_tenant" | "domain_mismatch" };

/**
 * TENANT_ALLOWLIST is a function (not a const) so it reads env vars at
 * call time. This matters for tests that set env vars per-case, and
 * avoids module-init-time coupling to env var loading order.
 */
export function TENANT_ALLOWLIST(): Record<string, TenantConfig> {
  const map: Record<string, TenantConfig> = {};
  const covermeshTid = process.env.ENTRA_COVERMESH_TENANT_ID;
  if (covermeshTid) {
    map[covermeshTid] = {
      emailDomain: "covermesh.com",
      displayName: "covermesh",
    };
  }
  // Future: Ecclesia, SCHUNCK — add ENTRA_ECCLESIA_TENANT_ID / _SCHUNCK_ env vars
  // and a block here. No other file needs to change.
  return map;
}

function domainOf(email: string): string | null {
  const at = email.indexOf("@");
  if (at < 0 || at === email.length - 1) return null;
  return email.slice(at + 1).toLowerCase();
}

export function isTrustedDomain(email: string): boolean {
  const domain = domainOf(email);
  if (!domain) return false;
  const map = TENANT_ALLOWLIST();
  return Object.values(map).some((t) => t.emailDomain === domain);
}

export function tenantForEmail(email: string): TenantConfig | null {
  const domain = domainOf(email);
  if (!domain) return null;
  const map = TENANT_ALLOWLIST();
  return Object.values(map).find((t) => t.emailDomain === domain) ?? null;
}

export function validateEntraSignIn(input: {
  tid: string;
  email: string;
}): EntraValidation {
  const { tid, email } = input;
  const map = TENANT_ALLOWLIST();
  const entry = tid ? map[tid] : undefined;
  if (!entry) return { ok: false, reason: "unknown_tenant" };
  const domain = domainOf(email);
  if (!domain || domain !== entry.emailDomain) {
    return { ok: false, reason: "domain_mismatch" };
  }
  return { ok: true };
}
```

```ts
// src/lib/auth/trusted-domains.ts
//
// Kept separate from entra-tenants.ts because this is imported by client
// components; entra-tenants.ts reads server-only env vars (tenant GUIDs).
// Updating this list and entra-tenants.ts together when onboarding a new

export const TRUSTED_DOMAINS = ["covermesh.com"] as const;

  if (at < 0 || at === email.length - 1) return false;
```

```ts
callbacks: {
  async signIn({ account, profile, user }) {
    if (account?.provider !== "microsoft-entra-id") return true;

    // Source the tenant id — Auth.js v5 surfaces it on profile as `tid`
    // for Entra. Fall back to decoding account.id_token if absent.
    let tid = (profile as { tid?: string } | null)?.tid ?? "";
    if (!tid && account.id_token) {
      try {
        const payload = JSON.parse(
          Buffer.from(account.id_token.split(".")[1], "base64url").toString("utf8")
        );
        tid = typeof payload.tid === "string" ? payload.tid : "";
      } catch {
        // fall through — empty tid triggers unknown_tenant below
      }
    }

    const email = user.email ?? "";
    const result = validateEntraSignIn({ tid, email });
    if (!result.ok) {
      console.warn(
        `[entra] reject sign-in: tid=${tid || "(missing)"} email=${email} reason=${result.reason}`
      );
      return false;
    }
    return true;
  },

  async session({ session, user }) {
    // …existing body unchanged
  },
},
```

```ts
// scripts/promote-to-admin.ts
// One-shot: promote a user to admin by email.
// Usage: npx tsx --env-file=.env.local scripts/promote-to-admin.ts user@covermesh.com
import { createClient } from "@libsql/client";

async function main() {
  const email = process.argv[2];
  if (!email) {
    console.error("Usage: promote-to-admin.ts <email>");
    process.exit(1);
  }

  const client = createClient({
    url: process.env.TURSO_DATABASE_URL ?? "file:./data/fakecarrier.db",
    authToken: process.env.TURSO_AUTH_TOKEN,
  });

  const before = await client.execute({
    sql: "SELECT id, email, role FROM users WHERE email = ?",
    args: [email],
  });
  console.log("before:");
  console.table(before.rows);

  if (before.rows.length === 0) {
    console.error(`No user with email ${email}`);
    client.close();
    process.exit(1);
  }

  const result = await client.execute({
    sql: "UPDATE users SET role = 'admin' WHERE email = ?",
    args: [email],
  });
  console.log(`rows affected: ${result.rowsAffected}`);

  const after = await client.execute({
    sql: "SELECT id, email, role FROM users WHERE email = ?",
    args: [email],
  });
  console.log("after:");
  console.table(after.rows);

  client.close();
}

main().catch((err) => {
  console.error(err);
  process.exit(1);
});
```

Operational steps for registering the FakeCarrier app in Entra and

1. **Certificates & secrets → Client secrets → New client secret.**
